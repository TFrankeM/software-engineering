Elicitação de Requisitos

Requisitos de Usuário

Requisitos Funcionais

  • O perfil do usuário consumidor deve conter foto, informações pessoais e histórico de compras, podendo ser aberto ou anônimo.
  • O aplicativo deve permitir vendas, promoções e recomendações personalizadas de produtos com base no perfil/histórico de compras do usuário.
  • Deve ser possível gerar relatórios de vendas, avaliações de produtos e estoque, que possam ser personalizados e exportados em formatos como CSV e Excel.
  • As máquinas devem estar conectadas ao aplicativo.
  • Caso o aplicativo apresente alguma falha, as máquinas não devem ser impactadas por ela.
  • Um sistema para notificar consumidores e vendedores sobre quando a máquina ficar indisponível.
  • O aplicativo deve permitir que usuários sigam perfis específicos de máquinas de venda para receber notificações sobre promoções e novidades.
  • Implementação de um programa de recompensas para fidelizar usuários que frequentemente compram através do aplicativo.

Requisitos Não-Funcionais

  • A interface do aplicativo deve ser simples e rápida, minimizando o tempo e as etapas necessárias para realizar uma compra.
  • O aplicativo deve estar acessível em diferentes locais da instituição, incluindo áreas com acesso externo liberado.
  • O sistema deve estar disponível a todos em horário acadêmico da instituição.
  • O funcionamento das máquinas não deve ser dependente do funcionamento do aplicativo.

Requisitos de Sistema

Requisitos Funcionais

  • O usuário terá a opção de adicionar, alterar e deletar sua foto de perfil, bem como suas informações pessoais em uma aba da aplicação.
  • Cada usuário deve poder visualizar em uma aba do aplicativo seu histórico de compras.
  • As máquinas de venda devem operar independentemente do aplicativo, garantindo que as vendas possam ser realizadas mesmo quando o aplicativo estiver offline.
  • O sistema deve fornecer funcionalidades para agendar manutenções e alertar administradores e usuários sobre máquinas fora de operação.
  • Em caso de falha de uma máquina, um alerta deve ser emitido imediatamente.
  • O sistema deve monitorar o estoque das máquinas em tempo real e alertar sobre condições que possam afetar a qualidade dos produtos.

Requisitos Não-Funcionais

  • Confiabilidade: O sistema deve garantir alta disponibilidade e funcionamento contínuo das máquinas de venda.
  • Segurança: Proteções adequadas devem ser implementadas para garantir a segurança das transações e dos dados dos usuários.
  • Escalabilidade: O sistema deve ser capaz de escalar conforme o aumento do número de usuários e máquinas sem degradação do desempenho.
  • O sistema deve estar disponível para todos durante o horário de funcionamento dos cursos da instituição.
  • Em caso de falhas não relacionadas à máquina, as vendas devem continuar normalmente, mas os dados não serão armazenados.
